---

# 🧪 🔄 Fluxo de QA com IA (passo a passo)

## 🧩 1. Recebeu a tarefa (Jira / user story)

👉 Antes de sair testando, faça isso:

**Use IA para entender melhor o requisito**

```text
Atue como QA sênior.

Analise esta user story:
[cole aqui]

Liste:
- dúvidas
- riscos
- cenários não cobertos
```

📌 Ferramenta ideal:

* Claude (melhor para análise profunda)

---

## 🔍 2. Quebrar em cenários de teste

Agora você transforma o requisito em testes:

```text
Crie cenários de teste completos para:

[funcionalidade]

Inclua casos positivos, negativos e edge cases.
```

📌 Resultado:

* você já sai com dezenas de cenários prontos

---

## ⚠️ 3. Pensar como QA sênior (onde vai quebrar?)

Aqui está o pulo do gato 👇

```text
Pense como um QA sênior.

Onde essa funcionalidade pode falhar?
Liste cenários críticos e erros ocultos.
```

👉 Isso evita bugs em produção (muito valorizado)

---

## 🧪 4. Executar testes manuais (com apoio da IA)

Durante os testes:

* peça ideias em tempo real
* explore além do básico

```text
Sugira testes exploratórios para:

[funcionalidade]
```

---

## 🐞 5. Encontrou bug? Gere o report perfeito

```text
Crie um bug report profissional com base nisso:

[descreva o erro]
```

📌 Ferramenta ideal:

* ChatGPT (mais direto e rápido)

---

## 🤖 6. Automatizar (se aplicável)

Transforme testes em código:

```text
Crie teste automatizado em [Playwright/Cypress/Selenium] para:

[cenário]
```

📌 Dica:

* ChatGPT → melhor pra gerar código
* Claude → melhor pra revisar/refinar

---

## 🔄 7. Regressão inteligente

Antes de entregar:

```text
Crie plano de regressão para:

[feature]

Priorize o que realmente precisa ser testado.
```

👉 Isso economiza MUITO tempo

---

## 📊 8. Aprender com o que aconteceu

Depois da task:

```text
Com base nesses bugs/testes:

[contexto]

O que poderia ter sido testado antes?
Como melhorar?
```

👉 Isso te faz evoluir rápido

---

# ⚙️ Fluxo resumido (pra salvar)

1. Entender requisito → IA analisa
2. Criar cenários → IA gera
3. Pensar falhas → IA ajuda
4. Testar → você executa + IA apoia
5. Reportar bug → IA melhora
6. Automatizar → IA acelera
7. Regressão → IA prioriza
8. Aprender → IA reflete

# 🚀 Exemplo real (mini cenário)

👉 Task: “Login com validação de senha”

Com IA você:

* descobre edge cases (senha vazia, caracteres especiais, limite, etc.)
* cria testes rapidamente
* gera automação
* escreve bug report profissional

Sem IA: 1h
Com IA: 15–20 min
