# PROMPT

No contexto de chatbots com IA, um **prompt** é simplesmente o **texto (ou instrução) que você envia para a IA** para obter uma resposta.

---

## 🧠 Definição simples

👉 **Prompt = o que você pede para a IA fazer**

Pode ser:

* uma pergunta
* um comando
* uma instrução detalhada

---

## 💡 Exemplos de prompt

* “Explique o que é QA”
* “Crie um teste automatizado em Selenium”
* “Resuma este texto”

Tudo isso são prompts usados em ferramentas como ChatGPT, Claude ou Gemini.

---

## ⚙️ Tipos de prompt

### 1. Simples

* “O que é API?”

### 2. Instrucional

* “Explique API como se eu fosse iniciante”

### 3. Contextual (mais avançado)

* “Sou QA iniciante, explique API com exemplos de testes”

👉 Quanto mais contexto você dá, melhor a resposta.

---

## 🎯 Por que o prompt é importante?

A qualidade da resposta depende muito do prompt.

👉 Um prompt ruim:

* “Explique isso”

👉 Um prompt bom:

* “Explique o que é teste de regressão com exemplos práticos em QA”

---

## 🧩 Conceito chave

Prompt está ligado a Processamento de Linguagem Natural, que permite à IA entender linguagem humana.

---

## 🧠 Resumão

👉 **Prompt é a forma como você se comunica com a IA.**
👉 Quanto melhor você escreve, melhor a IA responde.

# 🧪 Prompts para QA (Quality Assurance)

### 📌 1. Criar casos de teste

```
Você é um QA experiente.
Crie casos de teste para a seguinte funcionalidade:

[descreva a funcionalidade]

Inclua:
- cenário
- pré-condições
- passos
- resultado esperado
- casos positivos e negativos
```

---

### 📌 2. Encontrar cenários edge (muito importante)

```
Analise a funcionalidade abaixo e gere cenários de teste edge (casos extremos e erros):

[descrição]

Pense como um QA sênior e inclua riscos ocultos.
```

---

### 📌 3. Gerar testes a partir de user story

```
A partir desta user story, crie cenários de teste completos:

[cole a user story]

Considere critérios de aceitação e possíveis falhas.
```

---

# 🤖 Prompts para automação de testes

### 📌 4. Gerar código (Selenium / Cypress / Playwright)

```
Crie um teste automatizado usando [ferramenta] para o cenário:

[descreva o cenário]

Use boas práticas, organize o código e explique cada parte.
```

---

### 📌 5. Converter teste manual → automatizado

```
Converta este teste manual em um teste automatizado usando [ferramenta]:

[cole o teste]

Use padrão Page Object Model.
```

---

### 📌 6. Melhorar código de teste

```
Revise e melhore este código de teste automatizado:

[cole o código]

Aplique boas práticas, legibilidade e manutenção.
```

---

# 💻 Prompts para programação

### 📌 7. Explicação de código

```
Explique este código linha por linha de forma simples:

[cole o código]

Considere que sou iniciante.
```

---

### 📌 8. Debug (encontrar erro)

```
Analise este código e encontre possíveis erros ou bugs:

[cole o código]

Explique o problema e sugira correções.
```

---

### 📌 9. Criar função ou script

```
Crie uma função em [linguagem] que faça:

[descreva o objetivo]

Inclua tratamento de erros e exemplos de uso.
```

---

# 📚 Prompts para aprender mais rápido

### 📌 10. Aprender um conceito

```
Explique [tema] de forma simples, com exemplos práticos e analogias.

Depois, me dê um exercício para praticar.
```

---

### 📌 11. Simular entrevista

```
Simule uma entrevista para vaga de QA.

Faça perguntas uma por vez e avalie minhas respostas.
```

---

# 🚀 Prompt avançado (nível profissional)

Esse aqui é MUITO poderoso 👇

```
Atue como um QA sênior com experiência em automação.

Seu objetivo é:
- identificar riscos
- propor testes eficientes
- sugerir melhorias técnicas

Contexto:
[explique seu sistema ou tarefa]

Responda de forma estruturada e prática.
```

---

# 🧠 Dica de ouro (isso muda o jogo)

Ferramentas como ChatGPT, Claude e Gemini respondem MUITO melhor quando você usa:

👉 **papel + contexto + objetivo + formato**

Exemplo:

* papel: “QA sênior”
* contexto: “sistema de login”
* objetivo: “criar testes”
* formato: “lista estruturada”

---

## 🧩 Resumão

👉 Prompt bom = resposta profissional
👉 Prompt ruim = resposta genérica
